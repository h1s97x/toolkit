# 2026-06-22 Daily Log

## 建立包依赖关系

### 目标
让各包真正使用 `@h1s97x/errors` 和 `@h1s97x/logger`，建立正确的分层依赖。

### 依赖关系（最终状态）

```
@h1s97x/errors          (核心层，无依赖)
├── @h1s97x/logger       (基础层，依赖 errors)
├── @h1s97x/token-bucket  (基础层，无依赖)
│
├── @h1s97x/request-queue  (工具层，依赖 token-bucket + errors)
│
├── @h1s97x/api-client    (业务层，依赖 errors + logger)
└── @h1s97x/react-hooks   (业务层，依赖 errors + logger)
```

### 修改内容

1. **logger** (`packages/logger`)
   - `package.json` 添加依赖：`"@h1s97x/errors": "workspace:*"`
   - `src/logger.ts` 导入 `ConfigurationError, isBaseError, serializeError, toBaseError`
   - 构造函数验证 `level`，无效时抛出 `ConfigurationError`
   - `_log()` 正确处理 `BaseError`（保留原始实例 + 序列化字段）
   - transport 错误处理使用 `toBaseError`

2. **api-client** (`packages/api-client`)
   - `package.json` 添加依赖：`errors + logger`
   - `src/client.ts` 导入 `ExternalServiceError, createLogger`
   - `ApiError` 继承 `ExternalServiceError`
   - `fetchWithTimeout` 添加请求/响应日志
   - 错误统一用 `toBaseError` 包装

3. **request-queue** (`packages/request-queue`)
   - `package.json` 添加依赖：`errors`
   - `src/request-queue.ts` 导入 `RateLimitError`
   - 替换 4 处 `new Error(...)` 为 `new RateLimitError(...)`
     - 队列满 (`QUEUE_FULL`)
     - 超时 (`QUEUE_TIMEOUT`)
     - 清理 (`QUEUE_EVICTED`)
     - 手动清空 (`QUEUE_CLEARED`)

4. **react-hooks** (`packages/react-hooks`)
   - `package.json` 添加依赖：`errors + logger`
   - `src/use-api-fetch.ts` 导入 `isBaseError, toBaseError, createLogger`
   - 错误处理使用 `toBaseError`，日志使用 `createLogger`
